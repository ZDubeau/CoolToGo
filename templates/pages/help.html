Info de projet

  • Cette page permet de gérer les informations de connections à la base de donnée apidae. Il est possible de créer un projet et d'y associer les informations fournis sur le site Apidae: l'ID du projet et la clé API.Ces donées sont fournis par le site Apidae.
    La fonctionnalité lancement permet de récupérer les séletions qui ont été définies pour ce projet sur le site apidae et ensuite de lancer l'extraction de tous les lieux associés à chaque sélection.
    La fonctionnalité supprimer permet de supprimer un projet, les sélections pour ce projet et les lieux associés.

Gestion Selection Apidae

  • Cette page permet d'afficher les sélections associées aux projets ajoutés sur le site. Pour chaque sélection les informations du projet associé, du nom de la sélection sur le site Apidae, la date de la dernière extraction, le nombre de lieux extraits, le nom des catégories associées à la sélection sont affichées.
    La fonctionalité lancer permet de lancer la récupération des lieux associés à une sélection et les insérer dans la base de donnée du site.
    La fonctionnalité modifier permet de changer les catégories associées à une sélection. Attention les catégories ne seront mise à jour pour les lieux associés à cette sélection qu'au prochain lancement de l'extraction.
    Une tâche planifié qui se déclenche automatiquement à 3h toutes les nuits permet l'extraction des données pour toutes les sélections présentes dans la base de donnée.

Données de lieux

  • Cette page regroupe toutes les données de lieux dont le Front-End CoolToGo a besoin.
    Ces données sont regroupés commme suit :
    • - Apidae Extraction : permet d'afficher l'ensemble des lieux qui associés à des sélections qui ont été extrait depuis le site Apidae.
      Il est possible de modifier un lieu en choississant manuellement les catégories et les profils qui dovent lui être affecté.
    • - Entrées Manuelles : permet d'afficher l'ensemble des lieux qui ont créés manuellement sur ce site (le Back-End CoolToGo).
      Il est possible de choisir des catégories et profils différentes de celle initialement affectés et aussi de supprimer un lieu insérer manuellement.
    • - Toutes les données : permet d'afficher toutes les données de lieux quelles viennent du site Apidae ou qu'elles aient été créés manuellement.

Saisie Manuelle

  • Saisie manuelle permet d'ajouter un nouveau lieu sans qu'il ne viennent du site Apidae. Dans cette page il faut saisir l'ensemble des informations nécessaires pour le Front-End CoolToGo.

Outils d'associations

  • Cette section regroupe l'administration de plusieurs tables qui vont permettre d'automatiser la transformation des données extraites du site Apidae. C'est la pierre angulaire du traitement des données afin de limiter les retraitements manuels.
    • - Eléments de références : il s'agit d'une liste disponible sur le site Apidae et dont les éléments vont être utilisés dans chaque lieux de la base de donnée Apidae pour le caractériser.
      Par exemple, si un lieu est destiné à aux enfants, aux seniors, s'il est à proximité d'un cours d'eau, il y a plus de 5000 entrées permettant de caractériser un lieu.
      Cette fonctionnalité permet de charger le fichier Excel extrait du site Apidae ensuite il est possible d'associer ces éléments de référence à une catégorie, un profil ou un niveau de fraicheur.
      Il est possible de supprimer les éléments de réference non liés à une catégorie ou un profil ou un niveau de fraicheur et ceci pour éviter de saturer la base de donnée quand on utilise heroku en mode gratuit.
    • - Catégories : il s'agit des catégories qui seront utilisés sur le Front-End CoolToGo
      Il est possible d'ajouter une ou plusieur catégorie à une sélection.
      Il est possible de choisir de nouvelles catégories pour un lieu extrait de la base de donnée Apidae ou entré manuellement.
      Il est possible d'associer des éléments de références à une catégorie afin de déterminer la ou les catégories d'un lieu lors de la phase de transformation. Pour l'instant cette fonctionnalité n'est pas activé dans la transformation.
    • - Profils usager : il s'agit des profils usagers qui seront utilisés sur le Front-End CoolToGo
      Il est possible de choisir de nouveaux profils usagers pour un lieu extrait de la base de donnée Apidae ou entré manuellement.
      Il est possible d'associer des éléments de références à un profil usager afin de déterminer le ou les profil usager d'un lieu lors de la phase de transformation.
    • - Gestion de fraîcheur : il s'agit d'un moyen de caractériser le niveau de fraicheur d'un lieu.
      L'idée est d'associer des points en fonction des équipements présent dans les éléments de références d'un lieu et ainsi de pouvoir donner une "note de fraicheur".
      Lors de l'étape de transformation des données extraites d'Apidae, un score serait calculé à partir des points associées à chaque élément dans ce module.
      Pour l'instant cette fonctionnalité n'est pas disponible.

API

  • Cette page permet de tester l'une des fonctionnalités les plus importantes du projet voire même le but ultime : connecter le Front-End CoolToGo au Back-End CoolToGo.
    Les différents liens vont appeler l'API permettant au Front-End CoolToGo de récupérer les informations de catégories, de profils usages et de messages qui sont définies dans Back-End CoolToGo.
    Pour l"instant la fonctionnalité permettant de tester l'API pour les lieux (locations) n'est pas fonctionnelle par manque de temps pour intégrer du code AJAX dans cette page. C'est pourquoi il est conseillé d'utiliser Postman pour tester celle-ci.

Messages

  • La page messages permet de définir un ou des messages a afficher sur le Front-End de CoolToGo avec une date de début et une date de fin.
    La fonctionnalité modifier permet d'éditer le message.
    La fonctionnalité publier permet d'obtenir un json avec les informations du message.
    la fonctionnalité supprimer permet de supprimer le message.

Nouvel Administrateur

  • Cette page permet d'ajouter ou supprimer un Administrateur.
    Si aucun administrateur n'est présent dans la base de donnée, la page d'accueil permet de créer un nouvel administrateur.
